# 会社構造

## 概要

この会社は以下の4つの部署で構成されています。統括エージェントが各部署を調整し、タスクを適切に処理します。

---

## 部署一覧

### 1. タスク投入部署
- **役割**: ユーザーからタスクを受け取り、内容を分析して適切な部署に振り分ける
- **入力**: ユーザーからのタスク・依頼
- **出力**: 振り分け先部署への指示

### 2. タスク管理部署
- **役割**: タスクの進捗を管理し、期限と優先順位を整理する
- **管理場所**: `tasks/` フォルダ
- **入力**: 各部署からの進捗報告
- **出力**: タスク一覧・進捗レポート・優先順位リスト

### 3. リサーチ部署
- **役割**: 指定されたテーマを調査し、結果をファイルにまとめる
- **成果物保存先**: `research/` フォルダ
- **入力**: 調査テーマ・調査依頼
- **出力**: 調査レポート（Markdownファイル）

### 4. 資料作成部署
- **役割**: 指定された内容で資料・チェックリスト・マニュアルを作成する
- **成果物保存先**: `documents/` フォルダ
- **入力**: 作成依頼・テーマ・要件
- **出力**: 資料・チェックリスト・マニュアル（Markdownファイル）

---

## フォルダ構成

```
task-inbox/
├── CLAUDE.md          # 会社構造の定義（このファイル）
├── tasks/             # タスク管理部署の管理ファイル
├── research/          # リサーチ部署の成果物
└── documents/         # 資料作成部署の成果物
```

---

## ワークフロー

1. ユーザーがタスクを投入
2. **タスク投入部署**がタスクを分析し、担当部署を決定
3. 担当部署がタスクを実行
4. **タスク管理部署**が進捗・期限・優先順位を管理
5. 成果物を該当フォルダに保存し、ユーザーに報告
