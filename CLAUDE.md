# 会社構造

## 概要

この会社は4つの部署から構成されており、統括エージェントがすべての部署を管理・調整します。

---

## 部署一覧

### 1. タスク投入部署
- **役割**: ユーザーからタスクを受け取り、内容を分析して適切な部署に振り分ける
- **入力**: ユーザーからの指示・依頼
- **出力**: 振り分け先部署への指示

### 2. タスク管理部署
- **役割**: タスクの進捗を管理し、期限と優先順位を整理する
- **管理場所**: `tasks/` フォルダ
- **担当内容**: タスクの作成・更新・完了管理、優先順位付け、期限管理

### 3. リサーチ部署
- **役割**: 指定されたテーマを調査し、結果をファイルにまとめる
- **成果物保存場所**: `research/` フォルダ
- **担当内容**: 情報収集、調査レポート作成

### 4. 資料作成部署
- **役割**: 指定された内容で資料・チェックリスト・マニュアルを作成する
- **成果物保存場所**: `documents/` フォルダ
- **担当内容**: 資料作成、チェックリスト作成、マニュアル作成

---

## フォルダ構成

```
task-inbox/
├── CLAUDE.md        # 会社構造の定義（このファイル）
├── tasks/           # タスク管理部署が使用するタスク管理ファイル
├── research/        # リサーチ部署の調査結果ファイル
└── documents/       # 資料作成部署の成果物ファイル
```

---

## 業務フロー

1. ユーザーが統括エージェントにタスクを依頼する
2. **タスク投入部署**がタスクを受け取り、適切な部署に振り分ける
3. 振り分けられた部署（タスク管理・リサーチ・資料作成）がタスクを実行する
4. **タスク管理部署**が全体の進捗を監視・管理する
5. 統括エージェントがユーザーに結果を報告する
