# 会社構造

## 概要

この会社は4つの部署から構成され、統括エージェントがそれぞれの部署を管理・調整します。

---

## 部署一覧

### 1. タスク投入部署
- **役割**: ユーザーからタスクを受け取り、内容を分析して適切な部署に振り分ける
- **入力**: ユーザーからの指示・依頼
- **出力**: 各部署へのタスク割り当て

### 2. タスク管理部署
- **役割**: タスクの進捗を管理し、期限と優先順位を整理する
- **管理場所**: `tasks/` フォルダ
- **管理内容**: タスクのステータス、期限、優先順位、担当部署

### 3. リサーチ部署
- **役割**: 指定されたテーマを調査し、結果をファイルにまとめる
- **成果物保存場所**: `research/` フォルダ
- **出力形式**: 調査レポート（Markdownファイル）

### 4. 資料作成部署
- **役割**: 指定された内容で資料・チェックリスト・マニュアルを作成する
- **成果物保存場所**: `documents/` フォルダ
- **出力形式**: 資料・チェックリスト・マニュアル（Markdownファイル）

---

## フォルダ構成

```
task-inbox/
├── CLAUDE.md          # 会社構造の定義（このファイル）
├── tasks/             # タスク管理部署が使用するフォルダ
├── research/          # リサーチ部署の成果物フォルダ
└── documents/         # 資料作成部署の成果物フォルダ
```

---

## タスクの流れ

1. ユーザーがタスクを投入
2. **タスク投入部署**がタスクを分析し、適切な部署に振り分ける
3. **タスク管理部署**がタスクを登録し、進捗を追跡する
4. 担当部署（**リサーチ部署** または **資料作成部署**）がタスクを実行する
5. 成果物を該当フォルダに保存し、完了を報告する
