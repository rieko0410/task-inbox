# 会社構造

## 概要

この会社は統括エージェントが4つの部署を管理し、タスクを効率的に処理する組織です。

## 部署一覧

### 1. タスク投入部署
- **役割**: ユーザーからタスクを受け取り、内容を分析して適切な部署に振り分ける
- **入力**: ユーザーからの指示・依頼
- **出力**: 各部署へのタスク割り当て

### 2. タスク管理部署
- **役割**: タスクの進捗を管理し、期限と優先順位を整理する
- **管理場所**: `tasks/` フォルダ
- **入力**: 振り分けられたタスク一覧
- **出力**: 進捗レポート・優先順位リスト

### 3. リサーチ部署
- **役割**: 指定されたテーマを調査し、結果をファイルにまとめる
- **管理場所**: `research/` フォルダ
- **入力**: 調査テーマ・調査依頼
- **出力**: 調査結果レポート（research/ フォルダ内に保存）

### 4. 資料作成部署
- **役割**: 指定された内容で資料・チェックリスト・マニュアルを作成する
- **管理場所**: `documents/` フォルダ
- **入力**: 資料作成依頼・内容指定
- **出力**: 各種資料・チェックリスト・マニュアル（documents/ フォルダ内に保存）

## フォルダ構成

```
task-inbox/
├── CLAUDE.md        # 会社構造の定義（本ファイル）
├── tasks/           # タスク管理部署が使用するフォルダ
├── research/        # リサーチ部署が使用するフォルダ
└── documents/       # 資料作成部署が使用するフォルダ
```

## タスクフロー

1. ユーザーが統括エージェントにタスクを依頼
2. タスク投入部署がタスクを分析・振り分け
3. タスク管理部署が優先順位・期限を設定
4. 担当部署（リサーチ/資料作成）がタスクを実行
5. 統括エージェントが結果をユーザーに報告
